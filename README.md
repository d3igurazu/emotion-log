# emotion-log
GASとスプレッドシートを活用した、2次元グラフ表示機能付きの感情記録日誌（Emotion Journal）
"Initial commit: ユーザー認証UIの刷新、セキュリティ強化（PropertiesService対応）、および商用運用ロジックの基盤実装"

「感情記録日誌」開発・改善内容まとめ
1. ユーザーインターフェース（UI/UX）の刷新
認証画面の統合: ログイン画面と新規登録画面を一つのカードに統合。ボタン一つでスムーズに切り替えられる直感的なUIに改善しました。

パスワード表示制御機能: パスワード入力欄に「表示/隠す」ボタンを設置。入力内容を一時的に目視で確認できる機能を追加しました。

パスワード自動隠蔽（オートハイド）: セキュリティ強化のため、他の入力欄への移動や画面外のクリックを検知した際、表示中のパスワードを即座に伏せ字に戻すロジックを実装しました。

2. セキュリティとデータ保護
スクリプトプロパティによるID隠蔽: スプレッドシートID（機密情報）をコードから分離し、GASの「スクリプトプロパティ」へ移行。GitHub公開時に個人情報が漏洩しない安全な構造を確立しました。

接続テスト機能の実装: スクリプトプロパティに設定したIDが正しく動作するかをプログラムで検証するテスト用関数を構築しました。

3. バックエンド・データベースの最適化
doPostProxy（中継関数）の実装: フロントエンドからの複雑なリクエストを一括で受け取り、適切な処理（ログイン、登録、データ取得など）へ振り分ける堅牢な中継窓口を構築しました。

データベース自動初期化機能: プロジェクト開始時に必要なスプレッドシート（users / entries / keywords）を、プログラムが自動検知して作成する仕組みを実装しました。

4. 商用・運用向けロジックの導入
2次元感情マップの可視化: 入力された感情数値をSVG形式でグラフ化。ユーザーの状態を視覚的に把握できるマップ表示機能を実装しました。

試用期間およびアクティベーション機能: 登録から10日間の試用制限と、専用キーによるアクティベーション機能を搭載し、将来的な商用利用を見据えた機能を構築しました。
